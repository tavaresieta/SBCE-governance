# Brazilian Emissions Trading System (SBCE) — Governance & PB-MRV Coverage

An interactive, self-contained HTML infographic covering the **institutional governance
structure** of the Brazilian Emissions Trading System (SBCE) and the **phased sectoral
coverage** of the Brazilian MRV Program (PB-MRV).

The visual is split into two panels:

1. **Governance** — institutional org-chart showing the SBCE's deliberative, managing,
   advisory and external regulatory bodies, and the information flows between them.
2. **Rollout** — the PB-MRV sector-prioritization criteria, the staggered Stage 1–3
   entry timeline (2027–2035), and the phased implementation logic (Year 1 / Years 2 & 3 / Year 4).

## View it

- **Online (GitHub Pages):** see below.

## Host it on GitHub Pages

1. Push these files to a GitHub repository.
2. Go to **Settings → Pages**.
3. Under **Build and deployment**, set **Source** to `Deploy from a branch`, pick your
   `main` branch and the `/ (root)` folder, then **Save**.
4. After a minute your page is live at:
   `https://<your-username>.github.io/<repo-name>/SBCE%20Governance%20%26%20PB-MRV%20Coverage%20(embed).html`

> Tip: spaces and `&` in filenames become `%20` and `%26` in the URL. To keep links
> tidy, you can rename the file to something like `sbce-governance-pb-mrv.html` before uploading.

## Embed it in another page

Once the file is live, drop it into any web page or CMS (Notion, Confluence,
WordPress, Webflow, Google Sites…) with an iframe:

```html
<iframe
  src="https://<your-username>.github.io/<repo-name>/sbce-governance-pb-mrv.html"
  style="width:100%; height:2200px; border:0;"
  title="SBCE Governance & PB-MRV Coverage">
</iframe>
```

The document is responsive in width; adjust the `height` to fit your layout
(the merged visual is roughly 2150&nbsp;px tall).

## Editing

Open `SBCE Governance & PB-MRV Coverage.html` in any text editor. The two panels are
scoped under `.gov` (governance) and `.cov` (coverage) so their styles don't collide.
Box text lives directly in the HTML; the connector lines, timeline ticks and stage
positions are generated by small inline `<script>` blocks at the bottom of the file.

After editing the source, regenerate the self-contained build before re-hosting.
